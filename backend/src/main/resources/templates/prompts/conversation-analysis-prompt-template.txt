## 分析规则

### 1. 必须忽略的内容（不要出现在输出结果中）
- **口语重复词**：如 "i i i", "we we", "the the" 等重复
- **停顿词**：如 "um", "uh", "er", "ah" 等
- **语音识别错误**：无意义单词、乱码、识别错误的碎片
- **大小写问题**：如 "i" 应为 "I"，这是语音识别的常见问题，不算语法错误
- **标点符号问题**：口语转文字通常没有标点，不要分析

### 2. 分析目标
- 我已使用"用户"和"AI"标注出说话者，只分析用户说的句子，不分析 AI 的回答

### 3. 问题类型细分

#### 3.1 语法错误 (Grammar Error)
- `Tense`: 时态错误
- `Agreement`: 主谓一致
- `Plural`: 单复数错误
- `Article`: 冠词错误
- `Preposition`: 介词错误
- `Pronoun`: 代词错误
- `Structure`: 句式结构
- `Clause`: 从句错误
- `Word Form`: 词性错误
- `Comparison`: 比较级错误

#### 3.2 表达问题 (Expression)
- `Word Choice`: 用词不当
- `Collocation`: 搭配错误
- `Chinglish`: 中式英语
- `Redundancy`: 表达冗余
- `Tone`: 语气不当
- `Unnatural`: 表达生硬
- `Vocabulary`: 词汇贫乏
- `Formal`: 口语化不足

#### 3.3 其他
- `Incomplete`: 句子未完成或被打断
- `Chinese`: 中文表达

### 4. 分析要求（重要）
- **按句子合并分析**：一句话如果有多个错误，必须合并在一个对象中。
- **综合建议**：`suggestion` 必须是针对整句话的、最自然地道的完整表达。
- **错误列表**：在 `errors` 数组中列出该句中的每一个具体错误。

### 5. 输出格式（JSON）

**必须严格按照JSON Schema 输出结构输出**：

### 6. 示例

**输入**：
"Yeah, but your model have some several issues like the first, the first one you the first one."

**输出**：
```json
{
  "items": [
    {
      "originalSentence": "Yeah, but your model have some several issues like the first, the first one you the first one.",
      "suggestion": "Yeah, but your model has several issues. For example, the first one is...",
      "errors": [
        {
          "type": "Agreement",
          "point": "have→has"
        },
        {
          "type": "Redundancy",
          "point": "some several"
        },
        {
          "type": "Incomplete",
          "point": "句子重复且未完成"
        }
      ]
    }
  ]
}
```

---

**重要提醒**：
- 只输出 JSON，不要添加任何说明文字。
- 只包含有问题的句子，基本正确的句子不要输出。
- `suggestion` 应该是该情境下最地道的口语表达。
